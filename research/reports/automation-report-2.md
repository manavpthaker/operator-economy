# The Operator Economy: Tool & Architecture Decision Document for a Three-Gate Automated Production Flow

## TL;DR
- **Build the orchestration layer on self-hosted n8n as the front door, but hand long-running renders to Remotion Lambda (distributed) and keep pipeline state in Postgres — with n8n's native human-in-the-loop nodes routing all three approval gates to Slack/Gmail approve-reject buttons.** n8n runs everything in a single Node.js process by default and its `Wait`/human-in-the-loop nodes hold execution state across pauses — exactly what a solo operator needs for gates — but a 30-minute render must never block that process, so renders are fired to Lambda and polled.
- **The evidence-first brand is the single biggest automation risk.** YouTube's July 15, 2025 "inauthentic content" policy demonetizes mass-produced/templated content *channel-wide*, and the AI-disclosure flag (`status.containsSyntheticMedia`) is now settable via the API — so the guardrails that protect quality (human POV gate, materially varied substance, honest disclosure) are the same guardrails that protect monetization. Automate production plumbing; keep humans on judgment.
- **Total cost lands comfortably inside the $100–300/month budget:** roughly **$110–170/month at 1 video/week** and **$150–230/month at 2/week**, dominated by fixed SaaS subscriptions (ElevenLabs, a research API, email, thumbnails) rather than per-render compute, which is literally pennies.

## Key Findings

### 1. Orchestration layer
- **n8n (self-hosted) is the right front door for a solo operator.** Its human-in-the-loop feature was expanded in recent releases to route approvals through Gmail, Slack, Telegram, Discord, Microsoft Teams, WhatsApp, or a built-in chat, using a "Wait for Webhook" pattern: the workflow pauses at a decision point, sends an approve/deny (or editable custom form) to the operator, and resumes on click. The execution ID stays alive while waiting. This maps directly onto the three gates. Self-hosted community edition is free.
- **The critical weakness: n8n's default architecture runs the UI, webhooks, triggers, and workflow execution in a single Node.js process.** A 30-minute render inside an n8n node ties up that process and is fragile to restarts. The proven pattern is to **offload long jobs**: n8n fires the render to Remotion Lambda (distributed, chunked, no 15-min-timeout concern) or to a durable engine. A widely-referenced pattern hands the "hours/days" state to **Temporal** (durable timers, retries, idempotency) while n8n handles fast triggers and integrations. For a solo operator, the lighter-weight version is: keep state in n8n + Postgres, fire renders to Lambda, and poll.
- **Alternatives ranked:** **GitHub Actions** is excellent as the render/build runner (the pipeline is already Python + Remotion) and gives PR-style review gates and version-controlled artifacts, but it is a poor fit for indefinite human-wait states (jobs have time limits). **Make** is easy but a hosted black box with per-operation pricing that punishes multi-step workflows. **Temporal/cron + Python** is the most robust for durability and retries but the most engineering-heavy. **Claude Agent SDK / scheduled agent tasks** are strong for the research-and-draft *reasoning* steps but should not be the system-of-record orchestrator for a pipeline with renders and gates.
- **Human-in-the-loop patterns that work:** (1) native n8n approval nodes (zero extra cost, good for 1–3 workflows with a single reviewer — exactly this operator); (2) DIY webhook + Slack interactive buttons / n8n-hosted browser page; (3) a dedicated approval layer (gotoHuman, Humangent) only if you outgrow simple approve/reject. For this operator, **native nodes are the correct call** — adding another tool would not save time.

### 2. Research automation (topic → evidence brief)
- **For an evidence-first channel, Perplexity Sonar is the best fit for cited claims** because every response includes source URLs and supporting excerpts designed for verification. In an 8-question factual benchmark (LangSmith/LLM-as-judge), Perplexity API scored highest, with Exa second, Google Gemini Search Grounding third, and Tavily last — though the sample was tiny and directional. **Exa** wins on semantic/research discovery and `findSimilar` expansion; **Tavily** wins on cheap, structured, LangChain-native RAG snippets (1,000 free credits/month, then $0.008/credit).
- **Perplexity Sonar API pricing (2026):** `sonar` $1/$1 per 1M input/output tokens plus a per-request fee of $5/$8/$12 per 1,000 requests (low/medium/high search context); `sonar-pro` $3/$15 per 1M tokens plus $6/$10/$14 per 1,000 requests. **Rate limit: 50 requests/minute at Tier 0** (new accounts), scaling with lifetime spend. Tavily's free tier caps at 1,000 calls/month. Exa is ~$0.001–$0.005 per search.
- **Cost per weekly research run** (20–50 API calls across a couple of providers) is on the order of **$0.20–$2.00** — negligible. The binding constraint is not cost but **verification**: no search API "reliably produces cited claims" you can publish unchecked. Teams treat the API brief as *raw material* and verify at the human POV/rigor gate (Gate 1). The eval gates already in the pipeline should score claim-to-source linkage, not just craft.
- **Note on API churn:** Microsoft discontinued the Bing Search API in August 2025; Google sued SerpApi in December 2025; Nebius announced intent to acquire Tavily in February 2026; Brave retired its perpetual free tier in February 2026. Build a swap-out path; do not hard-couple to one vendor.

### 3. Asset generation
- **(a) Programmatic charts/motion:** **Remotion is the right choice** — you already run it, and it is the most production-ready for data-driven, React-component-based charts (drive all animation from `useCurrentFrame()`; disable third-party library animations to avoid render flicker; D3/SVG/HTML all work). Local render is roughly 1–5 min per minute of video; **Remotion Lambda parallelizes to ~30 seconds** by chunking across functions. **Motion Canvas** is better for hand-crafted educational animation but renders via Canvas 2D (no HTML/CSS) and has a smaller ecosystem; **Lottie** is for pre-built vector animations, not data-driven charts. Verdict: keep Remotion; add Lambda for cloud rendering.
- **Remotion Lambda cost is trivial:** the docs' own examples show a warmed render of the HelloWorld composition at ~$0.001, and a video-in-video composition at ~$0.017–$0.021; "most users render multiple minutes of video for just a few pennies." Full HD, <80-min videos are fine; the 10GB Lambda disk limits output to ~5GB / ~2 hours. **Watch two cost traps:** (1) S3 egress — assets pulled over HTTP incur egress even in-region, so host big assets on **Cloudflare R2** (zero egress); (2) the **Remotion license** — free for individuals/small companies, but render-based licensing applies above thresholds (Enterprise opt-out requires $500/month minimum). A solo operator under the small-business threshold renders license-free.
- **(b) B-roll APIs:** **Pexels and Pixabay** both offer free APIs under licenses that permit commercial, monetized YouTube use without attribution (Pexels License; Pixabay Content License). **The real risk is not the license text but three gotchas:** (1) neither platform *verifies* uploader ownership or indemnifies you — user-uploaded content can carry hidden third-party IP; (2) **no model/property releases** for recognizable faces or private property; (3) Content ID claims — Pixabay contributors can register free tracks/media with Content ID, so you can get a claim even on legitimately licensed media (a claim, not a strike — it can be disputed). For a monetized, evidence-first channel, prefer curated/vetted sources for anything with faces, and keep records of source + license + download date. **Storyblocks/Artgrid** are subscription-licensed and safer for commercial use but cost more and have thinner API stories.
- **(c) Thumbnail automation:** **Templated** ($29/mo for 1,000 renders, supports Canva import + REST + Zapier/Make/n8n/MCP) or **Bannerbear** ($49/mo Starter = 1,000 image credits; 1 image = 1 credit) are the branded-template-from-API workhorses. The **Canva API** is widely described as expensive/gated; the **Figma API** can render frames but is not built for high-volume templated output. **Automated 320px/180px legibility QA** is not a built-in feature of any of these — you build it: render the thumbnail, downscale to 320px and 180px with ffmpeg/sharp, and run a contrast/text-size (or vision-model) check before Gate 2. Verdict: **Templated or Bannerbear for generation; custom downscale-and-check for QA.**
- **(d) Playwright screen capture:** Playwright can drive a browser and record video of web-tool demos, and quality is viable for product demos (deterministic, scriptable, repeatable). **But this is the highest-maintenance automation in the stack** — selectors break when the target web app's UI changes, and a broken capture silently degrades a video. This is why the current pipeline correctly keeps **human-recorded screen captures at Gate 2**. Use Playwright only for stable, self-owned surfaces; keep humans for third-party tools that change UI.

### 4. Voice at scale
- **ElevenLabs remains the quality leader for narration** and is the right anchor. Per ElevenLabs' official API pricing page, API usage is billed in USD, not credits: **Text-to-Speech is $0.10 per 1,000 characters on Multilingual v2/v3, or $0.05 on Flash/Turbo.** At ~50 minutes/month of long-form VO (~60,000–90,000 characters), TTS runs about **$6–$9/month** on Multilingual v2/v3, or roughly half that on Flash (an independent 2026 calculator puts 10×5-min videos ≈ 60,000 chars at ~$12.36 on Multilingual v3 vs ~$6.18 on Flash v2.5). Eleven v3 is the newest, most expressive model; **pronunciation dictionaries (`.pls`, IPA/CMU) for finance terms work only on `eleven_flash_v2` and `eleven_v3`** — other models silently ignore phoneme tags and need alias workarounds. This is a real constraint: to get reliable pronunciation of finance jargon *and* dictionary support, standardize on v3 (or Flash v2) rather than Multilingual v2.
- **Character-timestamp API** (forced alignment / word-level timestamps) is available and used to sync Remotion captions/animations to VO.
- **Alternatives:** **Cartesia (Sonic-3)** leads on latency (~40ms TTFA) but latency is irrelevant for offline render — its edge is speed, not obviously higher narration quality. **OpenAI TTS** is convenient if already in the OpenAI stack but has no voice cloning and lower measured naturalness. **PlayHT was acquired by Meta (July 2025), rebranded PlayAI, and is reportedly being wound down — treat as deprecation risk.** For offline long-form where quality trumps latency, **ElevenLabs Multilingual v2/v3 (or a cloned brand voice on the same model) is the default.**
- **The "AI-fatigue" retention penalty is real for synthetic-sounding VO** and is the reason to (a) use the highest-quality model, (b) consider a cloned voice of the actual operator to keep it human-authored in spirit, and (c) invest in post-processing.
- **Post-processing:** normalize with a **two-pass ffmpeg `loudnorm`** chain (`loudnorm=I=-14:TP=-1.5:LRA=11` with `linear=true` and measured values fed back from pass 1) — single-pass "pumps" and is wrong for VOD. YouTube normalizes playback to ~-14 LUFS and only turns loud audio down (never boosts quiet), so target -14 (or -16, close enough). Add de-essing and section-boundary level matching. Verify output with the `ebur128` filter. `ffmpeg-normalize` (Python CLI) wraps all of this with built-in two-pass and batch support.

### 5. Publishing automation
- **YouTube Data API v3 got dramatically cheaper on December 4, 2025: `videos.insert` dropped from ~1,600 units to ~100 units per upload** — confirmed by Google's official YouTube Data API Revision History entry dated December 4, 2025, a 16× cut raising the free-tier ceiling from ~6 to ~100 uploads/day. Per Google's docs, projects get "a default quota allocation of 100 `search.list` calls, 100 `videos.insert` calls, and 10,000 units per day combined for all other endpoints" — `search.list` and `videos.insert` sit in *independent* 100/day buckets. For a 1–2 video/week channel this is a non-issue.
- **The AI-disclosure flag IS settable via API** — `status.containsSyntheticMedia` (boolean) on `videos.insert`/`videos.update`. This is a significant finding: you do not have to manually toggle "Altered or synthetic content" in Studio. Note YouTube's guidance: AI *production assistance* (scripts, outlines, thumbnails) does **not** require disclosure; realistic AI voice/likeness of a real person does. Per YouTube's own docs, disclosing does **not** limit reach or monetization — the label itself carries no penalty.
- **Scheduling works via API:** set `status.privacyStatus="private"` + `status.publishAt` (ISO 8601; only settable while private). **Thumbnails** via `thumbnails.set` (50 units); **playlists** via `playlistItems.insert` (50 units); **chapters** only indirectly, as timestamp lines in `snippet.description` (the API can write them; there is no dedicated chapters field).
- **What is NOT available via the API (Studio-only), confirmed:** **end screens, info cards, pinned comments, Test & Compare title/thumbnail A/B testing, and Shorts→long-form "Related Video" linking.** End screens/cards have an open Google feature request (issue #387277988) but no endpoint. This means the operator's Gate 3 preview approval must also cover the manual Studio steps (end screen, cards, pin comment, related-video link, kick off A/B test).
- **API-uploaded content and algorithmic treatment:** the evidence says **API uploads are treated the same as manual uploads by the recommendation algorithm** — the folklore of penalized API uploads is not supported. The real differences: (1) **unverified API projects created after July 28, 2020 upload as private by default** until you pass a one-time compliance audit — do this early; (2) since a March 2025 update, videos (even private) may get an additional human ad-suitability review taking up to 24 hours. Neither is an algorithmic demotion.
- **What multi-channel operators actually use:** a hybrid — direct API for programmatic upload + metadata; human-upload-with-prepared-metadata for the Studio-only finishing touches. Bulk tools (TubeBuddy/vidIQ) are used more for research/optimization than for upload. For this operator, **direct API upload + a human finishing pass at Gate 3 is the correct hybrid.**

### 6. Distribution automation
- **Email — beehiiv vs Kit (2026):** Kit raised prices ~35% in 2026. Kit's free "Newsletter" plan covers up to 10,000 subscribers (limited automation); beehiiv's free "Launch" covers up to 2,500. Under 5K subscribers, **both have viable free/cheap tiers.** **Recommendation: beehiiv** — stronger editor and growth/monetization tooling, 0% take on paid subscriptions (Kit takes a cut), better scaling pricing (beehiiv Scale ~$43/mo vs Kit ~$135/mo at 10K subs); both have APIs and integrate with Make/Zapier/n8n. **Choose Kit instead only if** advanced automation/funnels and selling digital products via email is core (Kit's visual automation builder is materially more powerful). Both APIs support creating drafts from markdown and per-video segments; note migrating *to* beehiiv does not carry over automations.
- **(b) Lead magnet landing page:** fastest conversion-optimized path is **beehiiv's or Kit's built-in landing pages + a welcome automation** (email-gated PDF/Notion delivery), or **Carrd ($19/year) + email-platform API** for maximum design control at trivial cost. Benchmark reality: the median landing page converts **6.6% across all industries** (Unbounce Q4 2024 Conversion Benchmark Report, 57M conversions across 41,000 pages and 464M visits; range ~3.8% SaaS to ~12.3% events). Crucially, **email-sourced traffic converts at ~19.3% median — nearly triple the all-industry median** (same report) — so newsletter/video-audience traffic is your best funnel. A tightly-scoped creator lead magnet with a clear outcome-based value prop can hit 30–40% (documented creator example: a "7 Days of AI" Carrd page at ~40% signup). The task's **0.5–2% of views → email capture** target is a realistic views-to-subscriber benchmark: at 2% of 10,000 monthly views you add ~200 subscribers/month. **Recommendation: beehiiv/Kit native landing page first; move to Carrd only if you need design control.**
- **(c) LinkedIn — this is the hard constraint.** LinkedIn's official API does **not** offer open personal-profile posting for third-party tools; document/carousel posting via API is not available for personal profiles. **In April 2025 LinkedIn ran an enforcement sweep against automation tools** (cookie-based auth, Chrome-extension overlays, engagement automation) — accounts were warned, restricted, and in cases permanently banned. During that sweep **Taplio's own company page was restricted**, and Taplio's own support docs state that Taplio X "is considered by default as an automation tool by LinkedIn, which goes against their Terms of Service." ToS-safe options: (1) **Buffer** (approved API access, safest for scheduling text posts); (2) tools built on the **official API** (e.g., Supergrow, MagicPost) rather than cookie/extension automation; (3) manual posting. **Avoid extension/cookie-based engagement automation entirely.** For an evidence-first personal brand, the account-loss risk isn't worth it — schedule via Buffer or post manually, and never automate engagement/DMs.

### 7. Analytics readback → self-correcting loop
- **Most of the channel's rubric-validation metrics are available programmatically via the YouTube Analytics API** (separate from the Data API; requires channel-owner OAuth). Available: **audience retention curve** (`audienceWatchRatio`/`relativeRetentionPerformance` across the `elapsedVideoTimeRatio` dimension — 100 data points per video, so you can read 30-second/intro retention), **card impressions/clicks/CTR**, **subscribers gained/lost**, **average view duration/percentage**, traffic-source breakdowns, demographics, and **returning viewers** (`newVsReturning`/`viewerType`). **Impressions and impressions-CTR by traffic source, plus end-screen element metrics, were officially added to the Analytics API on January 15, 2026** (`videoThumbnailImpressionsClickRate`, `endScreenElementImpressions/Clicks/ClickRate`) — historically Studio-only, so this is a recent and important unlock.
- **Caveats:** data has a **~48-hour processing delay** (retention typically 1–2 days), and some queries can be sampled/aggregated. Do not build tight real-time loops; a **weekly** readback matches the cadence.
- **Closed-loop content ops in practice:** the documented pattern is to read impressions→CTR→retention→returning-viewers as a *stack* (topic earns impressions, title/thumbnail earns the click, retention shows the promise was kept, audience data says what to make next), then reweight the topic-selection rubric. This is exactly what the operator's existing eval gates should consume: feed last week's per-video CTR-by-source and 30-second retention back into the research-run scoring so topic selection self-corrects. Doable today but few do it rigorously — a genuine edge for an evidence-first channel.

### 8. What NOT to automate
- **Do not automate away the human POV/rigor pass (Gate 1).** This is both a quality lever and a policy lever: YouTube's July 15, 2025 **"inauthentic content"** policy (renamed from "repetitious content") makes mass-produced/templated content ineligible for monetization **channel-wide** — one batch of formulaic videos can demonetize the whole channel. The defense is "materially varied substance" per video, which a human editorial pass guarantees.
- **Do not automate disclosure dishonestly.** Set `containsSyntheticMedia` accurately; since ~May 2026 YouTube's automatic detection can apply labels without consent and penalize repeat non-disclosure.
- **Do not automate LinkedIn engagement/DMs or use cookie/extension tools** (see §6c) — documented account bans.
- **Do not automate Playwright capture of third-party UIs you don't control** — maintenance debt exceeds time saved when selectors break silently.
- **Where lean operations keep humans:** editorial POV, factual verification of AI-compiled claims, thumbnail/title final judgment, and the preview approval. These are the four the current three-gate design already protects — the design is sound.

## Details: recommended end-to-end architecture

**Orchestrator:** self-hosted **n8n** (community edition, on a small VPS or home server) as the workflow spine and gate-notifier.
**State:** **Postgres** (n8n's own DB doubles as pipeline state); artifacts (scripts, VO, assets, renders) in **Cloudflare R2** (zero egress, feeds Remotion Lambda cheaply) or S3.
**Reasoning steps:** Python + Claude API (existing), invoked as n8n steps or via the Claude Agent SDK for research/draft stages.
**Render:** **Remotion Lambda** (fired from n8n, polled) — keeps 30-min renders off the n8n process.
**Gates → operator:** n8n native human-in-the-loop nodes → **Slack interactive buttons** (or Gmail approve/deny). Each gate posts the artifact + approve/reject/edit; execution pauses on the Wait node until the operator clicks.
**Publish:** YouTube Data API v3 (`videos.insert` with `containsSyntheticMedia`, `publishAt`, `thumbnails.set`) + a human Gate-3 finishing pass in Studio for end screens/cards/pinned comment/related-video/A-B test.
**Distribution:** beehiiv API (newsletter draft from markdown + lead-magnet automation); Buffer for LinkedIn; automated Shorts clip pipeline (existing).
**Readback:** weekly YouTube Analytics API pull → topic-scoring rubric reweight.

**Build order (4 weekly phases):**
- **Week 1 — Spine & Gate 1:** stand up n8n + Postgres + R2; wire research API (Perplexity Sonar) → Claude draft → existing eval gates → **Gate 1 Slack approval**. Get one script end-to-end to human approval.
- **Week 2 — Voice & assets & Gate 2:** ElevenLabs (v3, finance pronunciation dictionary) → two-pass ffmpeg loudnorm; asset plan → Templated/Bannerbear thumbnail + Pexels/Pixabay B-roll pull; **Gate 2 asset approval + human screen-capture handoff**.
- **Week 3 — Render & Gate 3 & publish:** Remotion Lambda render (assets from R2) → **Gate 3 preview approval** → YouTube Data API upload with disclosure flag + scheduled publish; document the manual Studio finishing checklist.
- **Week 4 — Distribution & readback loop:** Shorts/LinkedIn(Buffer)/beehiiv newsletter + lead-magnet landing page; wire weekly YouTube Analytics API pull into the topic-scoring rubric. Harden retries/idempotency.

## Details: monthly cost scenarios

**1 video/week (+3–5 Shorts, LinkedIn, newsletter):**
- VPS for n8n ~$5–15 · Remotion Lambda + R2 ~$1–5 · ElevenLabs (~50 min VO) ~$6–15 · Research API (light) ~$5–15 · Thumbnails (Templated) $29 · Email (beehiiv, <2.5K subs) $0 (Launch) or $43 (Scale) · Buffer LinkedIn $0–6 · Claude API (existing) ~$20–40.
- **Total: ~$70–130/month on free email tier; ~$110–170 with beehiiv Scale.** Comfortably within budget.

**2 videos/week:**
- Roughly doubles the *variable* costs (VO ~$12–30, research ~$10–25, render still pennies, more Claude tokens ~$40–70) while fixed SaaS (VPS, Templated, email, Buffer) stays flat.
- **Total: ~$150–230/month.** Inside the $300 ceiling with headroom.
- **Cost is dominated by fixed subscriptions, not per-render compute** — scaling cadence is cheap; the levers are the email tier and the research-API plan.

## Recommendations
1. **Week 1 – prove the spine.** Stand up self-hosted n8n + Postgres + R2 and get one video from research brief → Claude draft → eval gates → Gate 1 Slack approval. **Threshold to proceed:** a script clears the existing rigor eval and one human approval round-trips through Slack reliably.
2. **Weeks 2–3 – automate the middle, keep the gates.** Add ElevenLabs v3 (finance pronunciation dictionary) + two-pass loudnorm, Templated thumbnails with a custom 320/180px legibility check, Remotion Lambda rendering from R2, and YouTube Data API upload with `containsSyntheticMedia` set accurately and `publishAt` scheduling. **Threshold:** a full video reaches Gate 3 preview with <90 min total human time across all three gates.
3. **Week 4 – close the loop.** Wire the weekly YouTube Analytics API pull (retention curve, CTR-by-source now that impressions landed in the API Jan 2026, returning viewers) into the topic-scoring rubric. **Threshold to trust the loop:** two consecutive weeks where the rubric's topic ranking correlates with realized CTR×retention.
4. **Pass the API compliance audit early** so uploads aren't stuck private, and complete phone verification for custom thumbnails.
5. **Do the Studio-only finishing steps by hand at Gate 3** (end screens, cards, pinned comment, related-video link, kick off Test & Compare) — budget ~10 min; do not build automation for endpoints that don't exist.
6. **LinkedIn: schedule via Buffer or post manually. Never use cookie/extension automation.** Revisit only if LinkedIn opens official personal-profile posting.
7. **Benchmarks that would change the plan:** if research-API cost exceeds ~$30/mo, consolidate to one provider; if beehiiv's paid tier triggers before revenue, stay on free tier or Carrd; if any Playwright capture breaks twice, revert that demo to human capture permanently.

## Top 5 automation risks (evidence-first, human-authored brand) + guardrails
1. **Inauthentic-content demonetization (channel-wide).** *Risk:* templated/mass-produced output trips YouTube's July 15, 2025 policy and demonetizes the whole channel. *Guardrail:* keep the human POV/rigor gate; enforce "materially varied substance" per video via the eval rubric; never batch-publish formulaic episodes.
2. **Dishonest or missing AI disclosure.** *Risk:* auto-labeling wrong (either direction) erodes trust or triggers auto-labels/penalties. *Guardrail:* set `containsSyntheticMedia` deterministically from a per-video rule (realistic voice/likeness = yes; production assistance = no), reviewed at Gate 1.
3. **Unverified AI-compiled claims.** *Risk:* a hallucinated or mis-cited stat ships on an "evidence-first" channel. *Guardrail:* research APIs produce *raw material only*; Gate 1 verifies every load-bearing claim against its named source; eval gate scores claim-to-source linkage.
4. **LinkedIn account loss.** *Risk:* cookie/extension automation triggers restriction or permanent ban (April 2025 precedent). *Guardrail:* Buffer/official-API scheduling or manual posting only; zero engagement/DM automation.
5. **Silent asset/licensing failures.** *Risk:* a broken Playwright capture degrades a demo, or "free" B-roll carries hidden IP / a Content ID claim / no model release. *Guardrail:* keep human screen capture for third-party UIs; log source+license+date for every asset; prefer vetted sources for anything with recognizable faces; treat B-roll licensing as a review item, not a settled fact.

## Caveats
- **Vendor churn is high (2025–2026):** Bing Search API shut down (Aug 2025), PlayHT/PlayAI winding down, Tavily acquisition pending, Brave free tier retired, Kit price hike. Treat research and TTS providers as swappable; abstract them behind an interface.
- **The `videos.insert` ~100-unit figure reflects the December 4, 2025 quota cut** — older guides citing 1,600 units are stale. Quota is not the constraint for this cadence; the separate 100-call/day `search.list` cap could bite if research uses YouTube search heavily.
- **YouTube Analytics API has a ~48-hour delay and possible sampling**; the impressions/CTR-by-source and end-screen metrics only landed in the API on January 15, 2026, so third-party connectors may lag.
- **B-roll "free licenses" do not indemnify you** and lack model/property releases; Content ID claims can hit legitimately licensed media. For a monetized channel this is a legal-review item, not a settled fact.
- **The single largest brand risk is the tension between "automated" and "evidence-first, human-authored."** Every decision here is deliberately drawn to keep humans on judgment (POV, verification, final visual/title calls, preview) and machines on plumbing — because YouTube's inauthentic-content enforcement and audience AI-fatigue both punish the opposite choice.
- Some competitive/pricing figures come from vendor blogs and comparison sites; API pricing, quota, and policy claims are anchored to official Google/ElevenLabs/Perplexity documentation where cited, but verify live pricing before committing, as it moves quarterly.
