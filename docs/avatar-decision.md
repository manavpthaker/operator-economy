# Avatar decision ŌĆö digital twin corner block (2026-07-08)

## Decision

Every episode gets a face: Manav's HeyGen digital twin, lip-synced to the
section VO, composited as a corner block during direct-address sections
(hook + cta by default). Not full-runtime, not an on-camera recording per
episode, not a generic stock avatar.

## Why

- **Trust signal, not retention lever.** 2026 data: face vs. no-face does
  not move retention (50%+ AVD gets 3x recommendation regardless of
  format); what a face buys is the "real person" signal and comment
  engagement. For a channel whose moat is the operator POV pass, that
  signal is disproportionately valuable ŌĆö the 2026 inauthentic-content
  crackdown targets exactly the TTS-plus-slides pattern this pipeline
  could superficially resemble.
- **Scalability constraint is hard.** Per-episode camera time was ruled
  out (2026-07-08). The twin is recorded once (~5 min training footage),
  then every episode's clips are generated by the pipeline from the VO
  that already exists. Zero marginal camera time.
- **Corner block is where twins hold up.** Small block over schematic
  content reads convincingly; full-screen talking head is where the
  uncanny valley bites. Matches the "little block while screensharing"
  format anyway.

## Compliance

Same synthetic-media disclosure box as the ElevenLabs VO (checked at
upload, Gate 3 ŌĆö `compliance.ai_disclosure`). Own-likeness, disclosed,
delivering content anchored in real operator experience: inside the 2026
policy. The POV pass remains the authenticity substance; the avatar is
delivery surface.

## Implementation

- `config/blueprint.json` ŌåÆ `avatar{}` block: `enabled`, `avatar_id`
  (SET_ME until the twin exists), `segments` (section ids), dimension /
  background / corner / width_pct.
- `scripts/originate/generate_avatar.py` ŌĆö runs in `originate.py continue`
  right after `generate_vo.py`. Uploads each configured section's
  mastered `vo/<id>.mp3` to HeyGen, submits an audio-driven avatar job,
  polls, downloads `avatars/<id>.mp4`, writes `avatars/avatars.json`.
  Per-section resumable (jobs persist in `avatars/jobs.json`); safe under
  Cowork's 45s shell cap via repeated runs or `--submit-only`.
- `prepare_longform.py` ŌĆö attaches `avatar{src, corner, width_pct,
  margin_px}` to matching sections in render_data and owns the copy into
  `remotion/public/avatars/` (same never-stale rule as VO).
- `BlueprintComposition.tsx` ŌĆö `AvatarBlock`: absolutely-positioned
  corner block, hairline `ruleStrong` border on ink (Rev C schematic
  inset, no webcam bubble), sequenced over exactly the section's VO
  window, above scenes / below captions. **Muted** ŌĆö the section
  `<Audio>` stays the only audio source, so twin and VO cannot drift.

## Economics

Hook + cta Ōēł 35s/episode. HeyGen API ~$1-3/min ŌåÆ well under $2/episode.
Account has 1800 API credits (2026-07-08). Adding `thesis` (~75s) roughly
triples the cost; do it once the twin passes the eyeball test on a real
render.

## Engine revision ŌĆö photo mode is the default (same day)

Manav's review of the twin clip: mouth exaggerated, head/eyes static.
Root cause: instant-avatar motion is stitched from the training clip
(recorded fairly still), so the twin inherits the stillness and the
lip-sync over-articulates. Fix tested head-to-head from the same hook VO:

- **photo mode (WINNER, now default):** `POST /v2/videos` with
  `image_asset_id` (a real photo of Manav) + `audio_asset_id` +
  `motion_prompt` + `expressiveness: medium`. Avatar IV engine
  synthesizes natural head/eye/eyebrow motion from the audio. Real
  likeness (real photo), restrained mouth, controllable motion.
- **twin mode (fallback):** trained video avatar; footage motion. Kept in
  config; would improve with livelier re-recorded training footage.
- **look mode:** generated look image animated the same way as photo
  mode, but the face is "inspired by" Manav ŌĆö weaker likeness.

**The photo IS the set.** Wardrobe/backdrop in photo mode come from the
source photo, so the brand look = ONE deliberate photo (charcoal tee,
deep navy backdrop, warm key light, half-body, vertical), uploaded once
as `avatar.photo_asset_id`. The current asset is a frame pulled from the
twin clip (denim/home office) ŌĆö replace it when the brand photo exists.
Motion tuning knobs: `avatar.motion_prompt` (plain English) and
`avatar.expressiveness` (low/medium/high).

## Voice revision 2 ŌĆö room-matched ElevenLabs (FINAL, same day)

Second A/B settled it: the HeyGen clone's room quality won round one,
but the clone is muffled ŌĆö it was trained on the avatar footage's
distant camera mic, and no mastering restores what the mic never
captured. What actually won round one was the ROOM, not the voice. So:
`avatar.voice.source: "elevenlabs_room"` ŌĆö avatar sections lip-sync the
EL PVC VO processed through `room_chain` (subtle early reflections,
softened top, slight body, loudness re-anchor). Real voice likeness, the
tuned EL pipeline intact, native word timestamps valid (no Whisper, no
duration shifts ŌĆö EP002's storyboard timings hold), dry original kept at
`vo/<id>.dry.mp3`. The section below documents the heygen mode, which
remains a working fallback.

## Voice revision 1 ŌĆö HeyGen clone on avatar sections (superseded)

Manav's A/B (same photo, same hook script): the HeyGen voice clone
(oe-avatar-voice-01) beat the ElevenLabs studio VO for on-camera
sections ŌĆö "sounds like a person talking in a room," matching the face.
The clone was trained on the avatar footage's camera mic, so the room is
in the voice.

Mechanics (`avatar.voice.source: "heygen"`): avatar sections speak the
PERFORMED script (same performance pass, breath tags stripped) via the
clone; after download, `adopt_clip_audio()` extracts the clip audio,
masters it "rounder" (VOICE_MASTER_CHAIN ŌĆö gentle glue compression,
low-mid warmth, tamed top; NO de-esser or room-scrubbing, the room is
the point), tail-pads it, Whisper-aligns words, and overwrites the
section's vo/ caches. `originate.py continue` then re-runs generate_vo
(fast cache reassembly) so timeline/captions/music all follow the new
audio. Non-avatar sections stay ElevenLabs ŌĆö the room-vs-studio shift is
the documentary on-camera/narration convention.

Revert path: `avatar.voice.source: "elevenlabs_audio"` restores
lip-syncing the EL VO. EP002's original EL hook is backed up at
`vo/hook.elevenlabs.bak.mp3`.

## Looks (wardrobe + backdrop) ŌĆö added same day

The raw twin wears the training footage's clothes/room, which doesn't
match Rev C. HeyGen "looks" are prompt-generated wardrobe/backdrop
variants of the group, managed via `scripts/originate/avatar_looks.py`
(list / train / generate / status). `avatar.look_id` in config switches
clips to the look (photo-avatar engine); `generate_avatar.py --look`
overrides per episode.

Policy: ONE branded studio look as the default (charcoal/ink wardrobe,
deep navy backdrop echoing the schematic grid, warm cream key light) ŌĆö
consistent set, documentary read. Per-episode costume changes are
available but default to no: they read gimmicky and cost a look-generate
+ eyeball cycle per episode. CAVEAT: looks animate via the photo-avatar
engine, not the twin's real trained motion ŌĆö compare motion quality
against the trained-twin clip before adopting (both engines tested from
the same hook VO, 2026-07-08).
